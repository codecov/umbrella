{% load admin_urls %} {% comment %} Frequency chart for the celery_broker drill-down page. Shape: * `chart.queue_name` — the queue we materialised. * `chart.total_visible` — total messages in the snapshot the chart was computed from (`len(_fetch_all())`, NOT `LLEN`). * `chart.total_depth` — `LLEN(queue)` at the time the chart was built. When > `total_visible` we surface a "showing top N of M" banner so operators know the percentages are over the visible window. * `chart.buckets` — list of view-model dicts shaped as `{task_name, repoid, repo_display, repo_change_url, commitid, count, pct}`. Already sorted; rendered top-to-bottom. The `repo_display` / `repo_change_url` pair is computed in `_build_frequency_chart_context` via the same `_resolve_repo_displays` helper that powers the `repo_display` column on `RedisQueueAdmin`, so the chart's repo cell matches what operators see elsewhere. * `chart.clear_by_filter_url` — base URL for the row-level "Clear queue" form action; we POST to it with `task_name` / `repoid` / `commitid` / `csrfmiddlewaretoken`. The POST opens the preview page where the operator picks dry-run, clear-all- but-first, or clear-all. * `chart.can_clear` — boolean toggling the visibility of the "Clear queue" buttons (superuser-only; staff users see read- only percentages). {% endcomment %}
